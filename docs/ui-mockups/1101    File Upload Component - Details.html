Responsive Specifications

Desktop Layout

  • The component maintains a fixed aspect ratio with a clear dashed border to define the drop zone.
  • Text and icons are centered, providing ample whitespace.

Tablet Adaptations

  • The component scales down slightly but maintains its layout and aspect ratio.
  • Touch targets remain large and accessible.

Mobile Adaptations

  • The component takes up the full width of its container to maximize the drop zone area on smaller screens.
  • Padding is reduced to conserve vertical space.
  • Font sizes may be adjusted slightly based on the responsive typography scale.

Accessibility Features

Keyboard Navigation

Screen Reader Support

The label for the file input describes the drop zone's function. ARIA live regions (role='alert') are used to announce state changes like 'Upload complete' or 'Upload failed' to screen reader users.

Focus Management

A clear focus ring is applied to the 'Browse Files' and other interactive buttons within the component, consistent with the design system.

Aria Attributes

  • aria-live='polite' for status updates
  • role='alert' for error messages
  • aria-label for icon-only buttons

Color Contrast

All text and icons meet WCAG 2.1 AA contrast requirements against their backgrounds in all states (idle, success, error).
