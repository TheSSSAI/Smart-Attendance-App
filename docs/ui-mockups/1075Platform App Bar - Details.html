Description

The Platform App Bar serves as the primary header for application screens. It displays the screen title, navigation controls (like a back arrow or menu icon), and contextual action buttons.

Design System Alignment

Utilizes the design system's color palette for background and text, typography for the title, and spacing tokens for padding and layout. It adheres to the defined heights for touch targets and navigation elements.

Responsive Specifications

Mobile Adaptations

  • Height fixed at 56px to align with mobile platform standards.
  • Padding respects device notches and cutouts using `env(safe-area-inset-top)`.
  • Title is typically start-aligned after a leading navigation icon.

Tablet Adaptations

  • Generally follows the mobile pattern but with more space for action icons.
  • The 'web' variant may be used for larger tablets in landscape mode.

Desktop Enhancements

  • Height increased to 64px for better proportion on larger screens.
  • Increased horizontal padding for a more spacious layout.
  • Often has a subtle bottom border to separate it from the content area.

Accessibility Features

Keyboard Support

All interactive elements (icon buttons) within the app bar must be focusable and activatable via keyboard.

Screen Reader Support

The component is wrapped in a `

` tag. The title is a heading element (`

`). All icon buttons have accessible names via `aria-label`.

Focus Management

A logical tab order is maintained from left to right: leading icon, title (if link), action icons.

Aria Attributes

  • `aria-label` is used for all icon-only buttons.
  • Navigation icons (like a hamburger menu) should use `aria-haspopup` and `aria-expanded`.

Color Contrast

Text and icon colors meet the WCAG 2.1 AA requirement of 4.5:1 against the background color.
