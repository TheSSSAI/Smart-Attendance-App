Description

A responsive data table for displaying collections of data like users, teams, or reports. It transforms into a list of cards on mobile viewports to avoid horizontal scrolling and provide a better user experience. It supports loading, empty, error, and paginated states.

Design System Alignment

This organism component is composed of several molecular and atomic components from the design system. It uses Data Table Rows, Pagination Controls, and Loading Indicators. It heavily relies on the system's color palette for states, typography for content, and spacing tokens for layout.

Responsive Specifications

Mobile Adaptations

  • The entire table transforms into a list of cards.
  • Each table row becomes a distinct card.
  • Column headers are displayed as labels for each data point within the card using the `data-label` attribute for accessibility and context.

Tablet Adaptations

  • At 768px, the layout switches from cards to a traditional table view.
  • Uses CSS Grid for layout to ensure proper column alignment and responsiveness.

Desktop Enhancements

  • The table layout provides clear, scannable data presentation.
  • Hover states on rows provide visual feedback.
  • Column widths are defined using a flexible grid system to accommodate different data types.

Accessibility Features

Keyboard Support

Interactive elements within the table (checkboxes, buttons, pagination) are keyboard focusable and operable.

Screen Reader Support

On desktop, ARIA roles (`grid`, `row`, `columnheader`, `gridcell`) are used to create a semantic table structure. On mobile, `data-label` attributes ensure that screen readers announce the context for each piece of data (e.g., 'Status: Active').

Focus Management

All interactive elements have clear `:focus-visible` styles as defined in the design system.

Aria Attributes

  • role='grid'
  • role='row'
  • role='columnheader'
  • role='gridcell'
  • aria-label for icon buttons

Color Contrast

All text and status badges meet WCAG 2.1 AA contrast requirements.
