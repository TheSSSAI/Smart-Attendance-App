Description

A comprehensive form for creating or editing a single or recurring event. It includes fields for title, description, start/end times, recurrence rules, and assignment to individuals or teams.

Design System Alignment

This organism is composed of multiple molecular and atomic components from the design system, including Text Input Fields, Date/Time Pickers, Checkboxes, and Buttons. It adheres to the system's spacing, typography, and color tokens.

Responsive Specifications

Mobile Adaptations

  • The form is designed to be fully scrollable on small viewports.
  • Time fields stack vertically to improve usability on narrow screens.
  • Action buttons stack and become full-width for easy tapping.

Tablet Adaptations

  • The form maintains a two-column layout for time fields, taking advantage of the wider screen.

Desktop Enhancements

  • The form is typically displayed within a modal or a main content area with a max-width to ensure readability.

Accessibility Features

Keyboard Support

All form controls are keyboard-navigable in a logical order using the Tab key. Checkboxes can be toggled with the Space key.

Focus Management

All interactive elements have a clear, visible focus state consistent with the design system.

Aria Attributes

  • aria-describedby is used to link error messages to their respective inputs.
  • aria-busy='true' is applied to the form during a saving state to inform assistive technologies that the content is updating.

Color Contrast

All text, labels, and form control borders meet WCAG 2.1 AA contrast requirements.
