Responsive Specifications

Desktop Layout

  • The table is displayed within a card, utilizing the full available width.
  • All columns are visible, and content is clearly legible.

Tablet Adaptations

  • The layout remains the same as desktop, but horizontal scrolling may be triggered if the viewport width is less than the table's minimum content width.

Mobile Adaptations

  • A container with `overflow-x: auto` is used to allow horizontal scrolling of the table, ensuring no content is hidden or layout is broken. This is a pragmatic approach for complex data tables on small screens.
