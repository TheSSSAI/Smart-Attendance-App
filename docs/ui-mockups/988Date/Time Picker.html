Molecular Component: Date/Time Picker

A form control that allows users to select a date, time, or both, using a platform-native interface. This component is styled to look consistent with other text inputs but triggers the browser's or OS's built-in picker UI for a familiar user experience.

Component Variations

Variant Name: date_only

Used for selecting only a calendar date (year, month, day). Appropriate for scenarios like filtering reports by a specific day or setting an event date.

Variant Name: time_only

Used for selecting only a time of day (hours, minutes, and optionally AM/PM). Suitable for setting start/end times for daily work shifts or events where the date is already contextually understood.

Variant Name: platform_adaptive

Used for selecting both a date and a time together. This is the most common variant for scheduling specific appointments or events, such as 'January 15, 2025, at 10:30 AM'. The HTML `datetime-local` input type achieves this in a web context.
