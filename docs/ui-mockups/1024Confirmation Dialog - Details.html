Description

A modal dialog that interrupts a user's workflow to ask for confirmation before proceeding with an action. It's used for standard confirmations, destructive actions, and actions requiring a text input (like a justification).

Design System Alignment

This component utilizes the design system's color palette (primary, destructive, neutrals), typography for headers and body text, spacing scale for internal padding and margins, and border-radius/shadow tokens for the container.

Responsive Specifications

Mobile Adaptations

  • Dialog content width is fluid, respecting screen padding.
  • Action buttons stack vertically for easier touch interaction.

Tablet Adaptations

  • Dialog maintains its max-width.
  • Action buttons remain in a horizontal row.

Desktop Enhancements

  • Desktop layout is the same as tablet.
  • Benefits from hover states on action buttons.

Accessibility Features

Keyboard Support

The dialog container should trap focus. Tab and Shift+Tab cycle through focusable elements (buttons, inputs) within the dialog. Escape key should dismiss the dialog (equivalent to cancel).

Screen Reader Support

The dialog element should have `role="alertdialog"`, `aria-labelledby` pointing to the title, and `aria-describedby` pointing to the body content.

Focus Management

When the dialog opens, focus is automatically moved to the first focusable element (e.g., the Cancel button or the Confirm button).

Aria Attributes

  • role="alertdialog"
  • aria-modal="true"
  • aria-labelledby
  • aria-describedby

Color Contrast

All text and button contrasts meet WCAG 2.1 AA requirements.
