Description

A single row within a data table, designed to display a record's information across several columns. It supports various states like hover and selection, includes variants for different data types (users, teams, reports), and is responsive.

Design System Alignment

This component utilizes the design system's color palette for states (hover, selected), typography for cell content, and spacing for padding. It's built to contain atomic components like Checkboxes, Badges, and Icon Buttons.

Responsive Specifications

Mobile Adaptations

  • Less critical columns are hidden on viewports smaller than 768px.
  • A specific class `.data-table__cell--responsive-hide` is used to control visibility.

Tablet Adaptations

  • All columns are generally visible, but horizontal scrolling may be introduced for very wide tables.

Desktop Enhancements

  • All columns are visible.
  • Hover states provide clear feedback on interaction.

Accessibility Features

Keyboard Support

The row can receive focus using `focus-within` to highlight the active row. Interactive elements within the row (checkboxes, buttons) are individually focusable.

Screen Reader Support

Uses semantic `` and `` elements within a `

` with a proper ``. Checkboxes have associated labels.

Focus Management

A visible focus ring is applied to the row when a child element is focused, providing context.

Aria Attributes

  • The checkbox in a selected row should have `aria-checked='true'`.

Color Contrast

Text colors (`--color-neutral-700`) on background colors (`--color-neutral-white`, `--color-neutral-50`, `--color-primary-50`) meet WCAG AA requirements.
