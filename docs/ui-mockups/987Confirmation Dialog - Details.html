Description

A modal overlay that interrupts the user's workflow to confirm an action. It can be used for generic confirmation, warning about a destructive action (like deletion), or requiring user input before proceeding (like a rejection reason).

Design System Alignment

This component utilizes the design system's color palette (primary, neutral, semantic error), typography scale for titles and body text, spacing system for padding and layout, and integrates atomic button and input field components.

Responsive Specifications

Mobile Adaptations

  • Dialog content takes up a larger percentage of the screen width (e.g., 90-95%).
  • Buttons in the footer may stack vertically if horizontal space is insufficient.

Tablet Adaptations

  • Maintains a fixed max-width for the dialog content, ensuring it remains centered and does not stretch uncomfortably wide.

Desktop Enhancements

  • The overlay provides ample surrounding space, focusing attention on the dialog.
  • Ensures keyboard navigation and focus trapping (via JS, noted in accessibility).

Accessibility Features

Keyboard Support

The dialog container and all interactive elements (buttons, inputs) must be keyboard-focusable. The 'Escape' key should close the dialog. This requires JavaScript for full implementation.

Screen Reader Support

Uses `role='dialog'`, `aria-modal='true'`, `aria-labelledby` pointing to the title, and `aria-describedby` pointing to the main description text to provide context to screen reader users.

Focus Management

Focus must be programmatically moved to the dialog when it opens and trapped within it until it is closed. This is a critical feature that requires JavaScript.

Aria Attributes

  • role='dialog'
  • aria-modal='true'
  • aria-labelledby
  • aria-describedby

Color Contrast

All text and button colors meet WCAG 2.1 AA contrast requirements.
