Description

A self-contained component for displaying a single attendance record in a list format. It adapts its presentation for different user roles (Subordinate, Supervisor), supports multi-selection, and can display various status and exception flags.

Design System Alignment

This component utilizes the design system's color palette for semantic states (pending, approved, rejected), typography for hierarchy (name, date, times), and spacing tokens for internal layout. It also incorporates atomic components like Status Badge/Chip and Checkbox.

Responsive Specifications

Mobile Adaptations

  • Component adapts to 100% of its container's width.
  • Internal layout uses flexbox to wrap content gracefully on smaller screens or with larger dynamic text sizes.
  • Minimum touch target size of 48x48px is respected for the checkbox and the item itself.

Tablet Adaptations

  • No specific adaptations required; the component scales with its container.

Desktop Enhancements

  • Hover effects provide visual feedback on interactive elements.

Accessibility Features

Keyboard Support

When wrapped in a list, the component is focusable. If selectable, the checkbox can be toggled with the Space key.

Screen Reader Support

Semantic structure allows screen readers to announce all key information, such as 'Checkbox, unchecked. John Doe, January 20, 2025. Check-in 09:01 AM, Check-out 05:03 PM. Status: Pending.'

Focus Management

Focus indicators are handled by the browser or parent list component.

Aria Attributes

The checkbox input is properly associated with its visual representation.

Color Contrast

All text and badge colors are chosen from the design system to meet WCAG 2.1 AA contrast ratios.
