Description

The Bottom Navigation Bar is the primary navigation method for the mobile application. It provides persistent, ergonomic access to the main sections of the app for both Subordinate and Supervisor roles. It adapts its items based on the user's role and indicates the currently active screen.

Design System Alignment

This component uses the design system's primary and neutral color tokens for active and inactive states, caption typography for labels, and standard spacing tokens. It is designed to be placed at the bottom of the mobile viewport, respecting OS-level safe area insets.

Responsive Specifications

Mobile Adaptations

  • The component is designed exclusively for mobile viewports.
  • It uses `env(safe-area-inset-bottom)` to avoid being obscured by OS navigation elements (like the iOS home bar).
  • Flexbox layout ensures items distribute evenly across the available width.

Tablet Adaptations

  • This component is not intended for tablet or desktop layouts. A sidebar navigation would be used instead.

Desktop Enhancements

  • Not applicable.

Accessibility Features

Keyboard Support

The component is part of the standard tab order. Focusable links can be activated with Enter/Space.

Screen Reader Support

Uses a semantic `

` element. The active link is identified using `aria-current='page'`, allowing screen readers to announce 'current page'. Each link's text provides a clear destination.

Focus Management

Standard browser focus management applies. A visible focus indicator is provided for keyboard users.

Aria Attributes

  • aria-label='Main navigation' on the
    element.
  • aria-current='page' on the active link.

Color Contrast

Both active and inactive icon/text colors meet WCAG AA contrast ratios against the white background.
