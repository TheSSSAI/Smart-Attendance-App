Description

A high-level component that visually displays GPS check-in and check-out locations on a simulated map interface. It supports loading, error, and loaded states with single or dual pin variations. It is designed for mobile-first interaction.

Design System Alignment

This component utilizes the design system's color palette for pins and feedback states, typography for info windows, and spacing tokens for internal layout. It is a self-contained organism composed of smaller visual elements.

Responsive Specifications

Mobile Adaptations

  • The map view is fluid and takes up 100% of its container's width.
  • The `aspect-ratio` property ensures it maintains its shape across different screen sizes.
  • Interactions are optimized for touch via hover and focus-within states.

Tablet Adaptations

  • The component scales proportionally with its container. No specific adaptations are needed.

Desktop Enhancements

  • Hover states on pins provide immediate feedback for mouse users.
  • Keyboard navigation is supported by making pin containers focusable.

Accessibility Features

Keyboard Support

Each pin container is focusable via the Tab key. When focused, the info window appears.

Screen Reader Support

The main map container has an `aria-label` describing its purpose. Loading and error states use `role='status'` and `role='alert'` respectively to announce their content. The info window content is readable by screen readers when visible.

Focus Management

A clear, high-contrast focus ring is applied to the pin when it receives focus, making it obvious for keyboard users.

Aria Attributes

  • aria-label on the main container.
  • role='status' for loading state.
  • role='alert' for error state.

Color Contrast

Pin colors and info window text meet WCAG 2.1 AA contrast requirements against their respective backgrounds.

Implementation Notes

Embedded Css Architecture

All styles are embedded in the HTML head. CSS custom properties are used for design system tokens, and BEM is used for component class names to ensure modularity and prevent style conflicts.

Component Organization

The HTML file is structured as a showcase, demonstrating each state and variant of the component in a clear, labeled section. This allows for easy visual regression testing and serves as documentation.

Responsive Design

The component is built to be intrinsically responsive using modern CSS like `aspect-ratio` and `flexbox`, eliminating the need for specific breakpoints for this component.

Accessibility Compliance

Accessibility is implemented directly in the HTML and CSS. Pseudo-classes like `:focus-within` are used to create keyboard-accessible interactions without requiring JavaScript.

Browser Support

Designed for modern evergreen browsers. The map is a static image, ensuring it displays everywhere, while the interactive elements rely on standard CSS features.

Usage Guidelines

When To Use

Use this component on attendance detail screens to provide a visual representation of the check-in and check-out locations.

Content Guidelines

Info window content should be concise, typically showing the action type ('Check-In'/'Check-Out') and the corresponding timestamp.

Placement Rules

Typically placed within a card or detail view, often occupying a significant portion of the screen width to be easily viewable.

Customization Options

  • Use `.map-view--loading`, `.map-view--error`, or `.map-view--loaded` to set the component's state.
  • For the loaded state, position pin containers using `top` and `left` percentage values.
  • Use `.map-view__pin--check-in` or `.map-view__pin--check-out` for pin color variants.

Testing Recommendations

Browser Testing

Test on latest Chrome, Firefox, and Safari to ensure consistent rendering of CSS shapes and animations.

Device Testing

Test on mobile (iOS/Android) to verify touch interactions (simulated by hover/focus) and ensure the aspect ratio behaves as expected.

Accessibility Testing

  • Verify keyboard navigation by tabbing to each pin and confirming the info window appears.
  • Test with VoiceOver and TalkBack to ensure the map's label and the info window content are announced correctly.
  • Check color contrast of all text and pin colors.

Performance Testing

Ensure the pin drop-in animation is smooth and does not cause jank.
