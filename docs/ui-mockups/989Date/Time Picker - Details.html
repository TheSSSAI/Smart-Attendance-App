Description

A form control that allows users to select a date, time, or both using a platform-native interface. It is presented as a styled input field with a trigger icon, providing a familiar and accessible experience for users on their respective devices.

Design System Alignment

Utilizes design system tokens for colors (primary, neutrals, error), spacing, typography, and borders to maintain visual consistency with other form inputs like the Text Input Field.

Responsive Specifications

Mobile Adaptations

  • Leverages the native OS picker UI (e.g., Cupertino rollers on iOS, Material dialog on Android) provided by the browser for a familiar, touch-friendly experience.
  • The input field itself is responsive and adapts to the full width of its container.

Tablet Adaptations

  • Maintains the same behavior as mobile, presenting a large, easy-to-use native picker interface.

Desktop Enhancements

  • Uses the browser's desktop-native calendar/clock widget for selection, which is keyboard-navigable by default.

Accessibility Features

Keyboard Support

The input is focusable via the Tab key. The native picker UI is keyboard-navigable by default.

Screen Reader Support

Semantic `

Focus Management

A clear, high-contrast focus ring is applied to the input wrapper, consistent with other form elements.

Aria Attributes

  • `aria-describedby` is used to link the input to its corresponding error message.

Color Contrast

All text and border colors meet WCAG 2.1 AA contrast requirements.
