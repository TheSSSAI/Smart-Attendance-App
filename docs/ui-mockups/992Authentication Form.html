Component Variations

Variant Name : "login_email_password"

The primary form for users to log in using their registered email and password.

Variant Name : "login_phone_number_entry"

The initial step for passwordless login, where the user provides their phone number to receive an OTP.

The second step for phone login, where the user enters the 6-digit code they received via SMS.

Variant Name : "forgot_password"

A simple form for users who have forgotten their password to request a reset link via their email.

Variant Name : "set_new_password"

The final step in the password reset flow, where the user sets and confirms their new password after clicking the link from the email.
