Description

A card used in dashboards to display a key performance indicator (KPI), including a label, a primary value, and an optional change indicator. It supports a loading state using skeleton placeholders.

Design System Alignment

Uses design system tokens for color, typography, spacing, borders, and shadows to create a consistent data visualization element within the dashboard grid.

Responsive Specifications

Mobile Adaptations

  • Card takes full width of its container.
  • Typography scales down slightly if necessary, but primarily relies on wrapping.

Tablet Adaptations

  • Card fits within a multi-column grid layout, typically 2 or 3 columns.

Desktop Enhancements

  • Card fits within a multi-column grid layout, typically 3 or 4 columns.
  • Hover effects provide richer visual feedback.

Accessibility Features

Keyboard Support

The card itself is not interactive, but if it were wrapped in a link, it would be focusable.

Screen Reader Support

Semantic structure allows screen readers to announce the label, then the value, then the change indicator. For example: 'Total Users, 1,482. Up 12% from last month.'

Focus Management

Not applicable as a static component.

Aria Attributes

  • If the delta is significant, an `aria-label` on the delta element could provide more context, e.g., `aria-label='Increase of 12 percent'`.

Color Contrast

All text colors on the white background meet WCAG 2.1 AA requirements.
