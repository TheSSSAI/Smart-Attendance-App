Description

A modal dialog that interrupts the user's workflow to confirm an action, receive critical information, or provide input. It is used for standard confirmations, destructive actions, and actions requiring user input like a justification.

Design System Alignment

This component utilizes the design system's color palette for semantic states (primary, destructive), typography for titles and body text, spacing tokens for internal padding and layout, and relies on atomic button and text input components.

Responsive Specifications

Mobile Adaptations

  • Dialog width is set to 90% of the viewport to ensure it fits on small screens.
  • Max-height is set to 90vh, and the content area becomes scrollable if the message is long.

Tablet Adaptations

  • The dialog maintains its max-width of 450px, appearing as a centered modal.

Desktop Enhancements

  • The backdrop provides clear focus on the dialog, and keyboard navigation (Esc to close) is expected.

Accessibility Features

Keyboard Support

The dialog should be focus-trapped. The 'Escape' key should close the dialog. 'Tab' should cycle through focusable elements (close button, form inputs, action buttons).

Screen Reader Support

The component should use `

` with `aria-labelledby` pointing to the title and `aria-describedby` pointing to the content.

Focus Management

When the dialog opens, focus should be moved to the first focusable element inside it (e.g., the close button or a primary action button).

Aria Attributes

  • role="dialog"
  • aria-modal="true"
  • aria-labelledby
  • aria-describedby

Color Contrast

All text and button colors meet WCAG 2.1 AA standards for contrast.
