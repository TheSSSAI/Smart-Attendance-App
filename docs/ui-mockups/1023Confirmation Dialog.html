Molecular Component: Confirmation Dialog

A modal dialog used to confirm a user's intent before proceeding with an action. It comes in standard, destructive, and input-required variants.

Standard Confirmation Dialog

Used for non-destructive actions that require user confirmation, like canceling an unsaved form.

Discard unsaved changes?

You have unsaved changes in the event form. If you leave now, your changes will be lost.

Destructive Action Dialog

Used for actions that result in permanent data loss, such as deleting a team or user.

Delete Team 'Alpha Squad'?

Are you sure you want to delete this team? All associated member assignments will be removed. This action cannot be undone.

Dialog with Input

Used for actions that require mandatory user input for confirmation, like providing a rejection reason.

Reject Correction Request

Please provide a reason for rejecting this correction request. The reason will be visible to the employee.
