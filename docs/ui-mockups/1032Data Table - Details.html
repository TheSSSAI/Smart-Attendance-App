Description

A complex organism component for displaying collections of data in a structured, tabular format. It includes states for loading, empty, and error scenarios, and is composed of molecular (Data Table Row) and atomic (Pagination, Badges, Checkboxes) components. It is designed to be responsive via horizontal scrolling on smaller viewports.

Design System Alignment

This component heavily relies on the design system foundation, using tokens for typography, color, spacing, and borders. It orchestrates multiple smaller components, ensuring a consistent look and feel for all data presentation.

Responsive Specifications

Mobile Adaptations

  • The table container introduces horizontal scrolling on viewports where the table's min-width (600px) is greater than the viewport width.
  • Padding within cells is slightly reduced to conserve space.

Tablet Adaptations

  • The table may still require horizontal scrolling depending on the viewport width and orientation, but will generally show all columns.

Desktop Enhancements

  • All columns are fully visible without scrolling.
  • Hover effects provide clear feedback on row interaction.

Accessibility Features

Keyboard Support

The table itself is not interactive, but its contents (checkboxes, buttons in rows) are focusable and follow a logical tab order. The parent row receives a focus-within style for context.

Screen Reader Support

Focus Management

A visible focus ring appears on interactive elements. The entire row is highlighted when an element within it is focused.

Aria Attributes

  • aria-label for checkboxes and icon buttons to provide context (e.g., 'Select Jane Doe').

Color Contrast

All text and state colors (hover, selected) meet WCAG 2.1 AA contrast ratio requirements.
