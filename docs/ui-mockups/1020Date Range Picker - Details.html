Description

A form control that allows users to select a start and end date from a calendar popup. It combines two text inputs with a visual calendar interface, designed for filtering reports and forms.

Design System Alignment

Utilizes design system tokens for colors, spacing, typography, and borders. The input fields are styled according to the 'Text Input' atomic component specification, and buttons within the popup use the 'Primary Button' styles.

Responsive Specifications

Mobile Adaptations

  • On viewports smaller than 768px, the calendar popup detaches from the input and displays as a centered, fixed-position modal to provide a better touch-based user experience.

Tablet Adaptations

  • The component maintains its desktop layout, with the popup anchored to the input field.

Desktop Enhancements

  • The popup calendar is positioned directly below the input field, providing clear context. Hover states on calendar days provide interactive feedback.

Accessibility Features

Keyboard Support

The input fields are focusable via Tab. The calendar popup's interactive elements (navigation buttons, days, action buttons) are also tabbable.

Screen Reader Support

Semantic labels are provided for the 'Start Date' and 'End Date' inputs. The calendar grid uses ARIA roles (grid, gridcell) for proper navigation with screen readers. Selected dates are announced.

Focus Management

Focus is visually indicated on the entire component when either input is active. When the popup opens, focus should be programmatically moved into the calendar.

Aria Attributes

  • aria-label for calendar navigation buttons.
  • aria-selected for the selected dates in the grid.
  • aria-disabled for unavailable dates.

Color Contrast

All text and interactive elements adhere to WCAG 2.1 AA contrast ratios using the predefined design system colors.
