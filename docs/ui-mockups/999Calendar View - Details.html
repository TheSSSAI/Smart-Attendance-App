Description

An interactive calendar component for displaying events. It supports month, week, and day views, and is optimized for mobile touch interactions like swiping to navigate between periods. It includes states for loading, displaying events, and an empty state.

Design System Alignment

This component uses the design system's color palette (primary for selection/today, neutrals for text and borders), typography for dates and titles, and spacing tokens for layout. It integrates molecular components like Event List Item and atomic components like Status Badge/Chip.

Responsive Specifications

Mobile Adaptations

  • The calendar grid is fluid and takes up 100% of the container width.
  • Optimized for touch with large tappable day elements (44px+).
  • Navigation between months can be handled by swipe gestures (not implemented in this static CSS/HTML mockup).

Tablet Adaptations

  • The component maintains its mobile-first layout but may be placed within a larger two-column layout on a tablet screen.

Desktop Enhancements

  • Hover states on navigation buttons and calendar days provide visual feedback for mouse users.

Accessibility Features

Keyboard Support

All interactive elements (navigation buttons, days) are keyboard-focusable using `

Screen Reader Support

The calendar grid is wrapped in a container with `role='grid'`. Each day has `role='button'` and an `aria-label` providing full context (e.g., 'January 15, 2025, 2 events').

Focus Management

A clear, high-contrast focus ring is applied to all focusable elements.

Aria Attributes

  • role='grid'
  • aria-label for buttons
  • aria-current='date' for the current day
  • aria-selected='true' for the selected day

Color Contrast

All text and interactive elements meet WCAG 2.1 AA contrast requirements.
