Description

Provides a consistent, centered layout for unauthenticated pages like Login, Registration, and Forgot Password. It includes branding and a main content card area.

Design System Alignment

Uses design system tokens for background colors, spacing, shadows, and typography. It is designed to contain a 'Card' component as its primary content area.

Responsive Specifications

Mobile Adaptations

  • The main container is centered with padding.
  • The content card takes up most of the available width.
  • Vertical spacing is adjusted for smaller screens.

Tablet Adaptations

  • The layout maintains its centered nature, with the content card having a `max-width` to prevent it from becoming too wide.

Desktop Enhancements

  • The layout is optimized for a centered, focused experience, ideal for authentication forms.

Accessibility Features

Keyboard Support

The layout itself is not interactive, but all content placed within the card (forms, buttons) must be keyboard accessible.

Screen Reader Support

The layout uses a `

` element to denote the primary content area of the page, which is a best practice for accessibility.

Focus Management

Focus management is handled by the components placed within the layout's card.

Aria Attributes

    Color Contrast

    Default text and background colors are chosen from the design system to meet WCAG 2.1 AA standards.
