Molecular Component: Confirmation Dialog

A modal overlay used to confirm a user's intent for a significant action. It supports generic, destructive, and input-required variants.

Variant: Generic Confirmation

Used for standard, non-destructive confirmations.

Discard unsaved changes?

If you leave this page, any changes you've made will be lost. Are you sure you want to continue?

Cancel

Variant: Destructive Action

Used to confirm an irreversible action, such as deletion. The primary confirmation button uses the 'danger' style.

Delete Team 'Alpha'?

This will permanently delete the team and all associated data. This action cannot be undone.

Variant: With Input (Rejection)

Used when an action requires mandatory user input, such as providing a reason for rejecting a request.

Reject Correction Request?

Please provide a reason for rejecting this attendance correction request. The reason will be visible to the employee.
