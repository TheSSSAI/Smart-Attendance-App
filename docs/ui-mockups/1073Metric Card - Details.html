Description

A self-contained card used to display a key performance indicator (KPI) or a single, important data point. It typically includes a title, a large numerical value, and an optional trend indicator. It supports loading and error states.

Design System Alignment

This component utilizes the design system's color palette for backgrounds and semantic colors (success/error for trends), typography for titles and values, spacing system for internal padding, and shadow/border tokens for its card structure.

Responsive Specifications

Mobile Adaptations

  • The card takes the full width available within its grid container.
  • The large metric value font size may be slightly reduced on very small screens (< 360px) to prevent wrapping.

Tablet Adaptations

  • Maintains standard sizing and proportions within the responsive grid.

Desktop Enhancements

  • Hover effects (transform and shadow) are enabled to provide richer feedback.

Accessibility Features

Keyboard Support

The component is not interactive itself, but if it's wrapped in a link, it will be keyboard focusable.

Screen Reader Support

Semantic HTML structure allows screen readers to announce the title and value clearly. The loading state uses ARIA attributes to announce that content is loading. Trend indicators include visually hidden text for context.

Focus Management

Not applicable as the component is non-interactive.

Aria Attributes

  • aria-busy='true' on the loading state.
  • role='status' or 'alert' on the error state.

Color Contrast

All text and icons meet WCAG 2.1 AA contrast ratios against the white background.
