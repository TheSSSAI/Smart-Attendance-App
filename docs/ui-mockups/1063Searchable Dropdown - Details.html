Description

A form control that combines a text input for filtering with a dropdown list of options. It supports both single and multiple selections and is designed to be fully accessible and responsive.

Design System Alignment

Utilizes the design system's color palette for states (primary for focus, error for validation), typography for text, spacing for padding, and shadows for the dropdown menu's elevation.

Responsive Specifications

Mobile Adaptations

  • On mobile viewports (< 768px), the dropdown list transforms into a full-screen modal upon opening.
  • The modal contains a header with the search input and a close button for a mobile-friendly experience.
  • The list of options becomes scrollable within the modal body.

Tablet Adaptations

  • Maintains the desktop dropdown behavior but ensures touch targets for options are sufficiently large.

Desktop Enhancements

  • Traditional dropdown menu behavior that opens below the input.
  • Supports keyboard navigation (up/down arrows, Enter, Escape).
  • Focus remains on the input field, allowing continuous typing while the list is open.

Accessibility Features

Keyboard Support

Full keyboard navigation is intended: Arrow keys to navigate options, Enter to select, Escape to close the dropdown.

Screen Reader Support

Uses a composite `combobox` ARIA pattern. The input has `role='combobox'`, `aria-controls` links it to the `role='listbox'` container. Options have `role='option'` and `aria-selected` state.

Focus Management

Focus remains on the input field when the dropdown is open. Tabbing from the input should close the dropdown and move to the next form element. On mobile, focus moves to the input inside the modal.

Aria Attributes

  • aria-haspopup="listbox"
  • aria-expanded="true/false"
  • aria-controls="[listbox_id]"
  • aria-labelledby="[label_id]"
  • aria-activedescendant="[focused_option_id]" for keyboard navigation

Color Contrast

All text and state colors meet WCAG 2.1 AA contrast ratios against their backgrounds.
