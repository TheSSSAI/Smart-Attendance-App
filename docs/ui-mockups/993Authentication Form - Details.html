Description

A comprehensive organism for handling all user authentication flows, including login with email/password, phone OTP, password reset, and new user registration. It combines atomic components like Text Input Fields, Buttons, and Links into a cohesive, stateful form.

Design System Alignment

This component heavily relies on the design system's foundation for typography, colors, spacing, and form controls. It assembles atomic components (Buttons, Inputs, Links) and feedback components (Alerts, Spinners) into a functional user-facing organism.

Responsive Specifications

Mobile Adaptations

  • The form uses a single-column, vertical layout which is ideal for mobile screens.
  • Inputs and buttons are full-width to be easily tappable.
  • The layout is designed to accommodate the on-screen keyboard by allowing the view to scroll.

Tablet Adaptations

  • The form maintains its `max-width`, appearing as a centered card on larger screens.

Desktop Enhancements

  • The form is displayed as a card in the center of the viewport, maintaining a focused experience.

Accessibility Features

Keyboard Support

All form elements are keyboard-navigable in a logical order using the Tab key. Actions can be triggered with Enter/Space.

Screen Reader Support

Semantic HTML (`form`, `label`, `button`) is used. Error messages are linked to inputs with `aria-describedby`.

Focus Management

Inputs have a clear, high-contrast focus ring as defined in the design system.

Aria Attributes

  • aria-describedby
  • aria-invalid
  • role='alert' for the error banner

Color Contrast

All text and control colors meet WCAG 2.1 AA standards.
