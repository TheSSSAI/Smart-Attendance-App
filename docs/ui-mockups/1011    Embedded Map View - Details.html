Responsive Specifications

Desktop Layout

  • Component scales to fit the container it's placed in. Maintains aspect ratio.
  • Hover interaction on pins reveals info windows.

Tablet Adaptations

  • Same as desktop layout, with larger touch targets for pins.

Mobile Adaptations

  • Layout is optimized for vertical scrolling contexts. Tap interaction on pins reveals info windows.
  • The map view itself is pannable and zoomable with standard touch gestures.

Accessibility Features

Keyboard Navigation

Each pin should be a focusable element in the tab order.

Screen Reader Support

The map container should have an ARIA label like 'Map showing attendance locations.' Each pin should have a descriptive label, e.g., 'Check-in location pin. Press to see details.' Info windows are associated with their respective pins.

Focus Management

When an info window opens, focus should ideally move to it. This requires JavaScript.

Aria Attributes

  • role='group'
  • aria-label
  • aria-describedby

Color Contrast

Pin and info window colors are chosen from the design system's primary and semantic palettes to ensure they have sufficient contrast against a typical map background. Text within info windows meets WCAG 2.1 AA standards.
