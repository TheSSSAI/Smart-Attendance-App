Molecular Component: Bottom Navigation Bar

The primary navigation component for the mobile application, providing access to top-level destinations.

Subordinate Variant

Home Dashboard

This is the main content area for the Subordinate user. The navigation bar below remains fixed at the bottom of the screen.

  • Home
  • Calendar
  • History
  • Profile

Supervisor Variant

Team Approvals

The Supervisor's view has a different set of navigation items, prioritizing team management tasks.

  • Approvals
  • Team
  • Schedule
  • Profile

Implementation & Usage

The Bottom Navigation Bar is implemented using a semantic <nav> element with a list of links. It is designed to be fixed to the bottom of the viewport.

Key Features

  • Role-Based Variants: The items in the navigation bar change depending on whether the user is a 'Subordinate' or 'Supervisor'.
  • Active State: The currently viewed screen is indicated with a primary color and bolder text, using the .bottom-nav__link--active class and aria-current="page" for accessibility.
  • Safe Area Aware: The component includes padding to account for the gesture navigation bar on modern iPhones and Android devices using env(safe-area-inset-bottom).

Component Variations

Variant Name : subordinate_nav

Navigation for the Subordinate user role, focusing on personal attendance and scheduling.

Variant Name : supervisor_nav

Navigation for the Supervisor user role, prioritizing team management and approval tasks.
