Component Variations

Variant Name : single_event

The default form for creating a one-time, non-recurring event.

Variant Name : recurring_event

An expanded version of the form that appears when the 'Repeat Event' option is selected. It includes controls for frequency (daily, weekly, monthly), days of the week, and a recurrence end date.

Variant Name : default

The standard interactive state of the form, ready for user input.

Variant Name : validating

A state showing client-side validation errors, such as a missing title or an end time that occurs before the start time. The 'Save' button is disabled, and clear error messages guide the user.

Variant Name : saving

A state indicating that the form has been submitted and is awaiting a response from the server. All form fields are disabled, and the primary action button shows a loading state to prevent duplicate submissions.
