Description

A form control component that allows users to select a start and end date. It combines two text inputs with an interactive calendar popup, which adapts for mobile and desktop views.

Design System Alignment

This component utilizes the design system's color palette for borders, focus states, and calendar elements. It inherits typography, spacing, and form input styles from the atomic Text Input Field, ensuring a consistent look and feel.

Responsive Specifications

Mobile Adaptations

  • On screens narrower than 768px, the calendar popup transforms into a full-screen modal with a backdrop.
  • The modal presentation is optimized for touch, providing a larger, more focused interface for date selection.
  • Calendar day touch targets are slightly increased for better usability on mobile.

Tablet Adaptations

  • Maintains the desktop-style dropdown popup for the calendar, as screen real estate is sufficient.

Desktop Enhancements

  • The calendar is presented as a dropdown popup positioned below the input fields.
  • Utilizes `:focus-within` to provide a clear visual grouping and focus state for the entire component when either input is active.

Accessibility Features

Keyboard Support

The component can be activated using keyboard navigation (Tab to focus, Space/Enter to open calendar). Calendar navigation would require JS, but this CSS-only mockup ensures the trigger is accessible.

Screen Reader Support

Each input field is associated with a `label` for screen readers. The trigger label encompasses both inputs to indicate a single interactive element.

Focus Management

A clear, high-contrast focus ring is applied to the entire component container using `:focus-within`, providing context to keyboard users.

Aria Attributes

    All text and interactive elements are designed to meet WCAG 2.1 AA contrast requirements.
