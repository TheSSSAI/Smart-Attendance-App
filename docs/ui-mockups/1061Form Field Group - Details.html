Description

A molecular component that groups a label, an input field, and an optional validation message to create a complete, accessible form element.

Design System Alignment

Utilizes design system tokens for spacing (gap), typography (label font weight, error message size), and colors (text, borders, error state).

Responsive Specifications

Mobile Adaptations

  • The component is fluid and takes up 100% of its container width by default.
  • Labels and validation messages wrap to multiple lines as needed to accommodate dynamic text scaling and smaller viewports.

Tablet Adaptations

  • No specific adaptations needed; responsive behavior is inherited.

Desktop Enhancements

  • On larger screens, the component can be placed within grid layouts to control its maximum width.

Accessibility Features

Keyboard Support

The nested input element is fully focusable and interactive via keyboard.

Screen Reader Support

The `

Focus Management

Focus styles are handled by the atomic Text Input Field component.

Aria Attributes

  • `aria-describedby` is used to link the input field to its validation message, ensuring screen readers announce the error.

Color Contrast

Label text, input text, and error text colors meet WCAG 2.1 AA contrast ratios against their backgrounds.
