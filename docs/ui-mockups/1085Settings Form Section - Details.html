Component Name

Settings Form Section

Component Type

organism

Description

An organism component that provides a structured and consistent layout for a section of a settings page. It includes a title, description, content area for form controls, and a footer for actions. It supports different variants for different types of settings, such as standard configuration and destructive 'danger zone' actions.

Design System Alignment

This component is composed of molecular components (Form Field Group) and atomic components (Primary Button, Toggle Switch), all styled using the foundational design system tokens for color, typography, spacing, and borders.

Responsive Specifications

Mobile Adaptations

  • The section's form elements stack vertically.
  • The label for a form row appears directly above its corresponding control.

Tablet Adaptations

  • Follows the desktop layout.

Desktop Enhancements

  • On screens wider than 768px, the form rows adopt a two-column layout, with the label and description on the left and the form control on the right, providing a more spacious and scannable interface.

Accessibility Features

Keyboard Support

All interactive elements within the section (inputs, toggles, buttons) are keyboard focusable and operable.

Screen Reader Support

The section title is a heading (`

`). Each form control is properly associated with its label via `

Focus Management

A logical tab order is maintained through the form controls and footer actions. Focus states are clearly visible.

Aria Attributes

  • `aria-describedby` is used within `Form Field Group` to link error messages to inputs.

Color Contrast

All text and UI elements meet WCAG 2.1 AA contrast requirements.
