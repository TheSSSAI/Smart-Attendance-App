Description

An organism component for handling file uploads. It supports drag-and-drop and button-based file selection, and displays various states such as idle, uploading, complete, and error.

Design System Alignment

This component utilizes the design system's color palette for states (primary, error), typography for instructional text, spacing for layout, and depends on atomic components like Primary Button and Loading Indicator.

Responsive Specifications

Mobile Adaptations

  • The drag-and-drop zone maintains a large, easily tappable area.
  • Text and icons scale appropriately with dynamic text settings.
  • The component is fluid and adapts to the container's width.

Tablet Adaptations

  • No specific adaptations needed; the component's responsive nature handles tablet sizes gracefully.

Desktop Enhancements

  • A distinct hover state on the drop zone provides clear feedback for drag-and-drop interactions.

Accessibility Features

Keyboard Support

The hidden file input is associated with a label, making the entire drop zone focusable and activatable with a keyboard (Enter/Space).

Screen Reader Support

The drop zone is announced as a file input. State changes (uploading, complete, error) are contained in elements with ARIA live regions or roles like 'alert' to be announced.

Focus Management

Focus indicators are applied to the 'browse' link and any action buttons.

Aria Attributes

  • aria-live='polite' for progress updates.
  • role='alert' for error messages.
  • aria-labelledby on the file input to connect it to the main text.

Color Contrast

All text and semantic colors for states meet WCAG 2.1 AA contrast ratios.
