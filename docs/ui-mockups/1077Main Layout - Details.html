Description

The foundational structure for the application, organizing the main navigation (sidebar or bottom nav), header, and content areas. It adapts its layout based on the platform (web vs. mobile).

Design System Alignment

This component orchestrates the entire screen, using design system tokens for background colors, spacing for margins and padding, and typography for content. It serves as the primary container for molecular components like the Platform App Bar and navigation elements.

Responsive Specifications

Mobile Adaptations

  • Layout is a flex column with a fixed top app bar and a fixed bottom navigation bar.
  • The content area scrolls independently between the header and footer.
  • Sidebar is not present in this variant.

Tablet Adaptations

  • The web layout is used, typically in its 'collapsed' state for portrait orientation and 'open' state for landscape.
  • The sidebar may transform into an overlay/drawer on smaller tablets, requiring JavaScript for state management.

Desktop Enhancements

  • Uses a CSS Grid layout to create a persistent sidebar and a main content area.
  • Supports 'open' and 'collapsed' states for the sidebar, controlled by a class on the root layout element.
  • Sidebar is sticky, allowing content to scroll independently.

Accessibility Features

Keyboard Support

The layout structure ensures a logical tab order: sidebar navigation, then main content. All interactive child components (navigation links, buttons) are keyboard accessible.

Focus Management

Focus indicators are handled by the child components (buttons, links). When the sidebar collapses, focus remains logical.

Aria Attributes

  • `role='navigation'` and `aria-label` for sidebar and bottom navigation to distinguish them.
  • Sidebar toggle button uses `aria-expanded` and `aria-controls`.

Color Contrast

All background and text color combinations adhere to WCAG 2.1 AA contrast requirements.
