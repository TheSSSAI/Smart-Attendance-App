Description

A full-screen component for Supervisors to view and manage pending attendance or correction requests from their subordinates. It supports loading and empty states, as well as a multi-selection mode for bulk actions.

Design System Alignment

This organism is composed of molecular components like the Contextual App Bar and Attendance List Item, and atomic components like Checkbox and Primary Button. It adheres strictly to the design system's tokens for color, typography, and spacing.

Responsive Specifications

Mobile Adaptations

  • The component is designed as a full-screen mobile view.
  • The list content is vertically scrollable.
  • The App Bar adapts to device notches and status bars.

Tablet Adaptations

  • Not applicable. This component is designed for a mobile-first, phone-sized viewport.

Desktop Enhancements

  • Not applicable. This component is part of the mobile application, not the web dashboard.

Accessibility Features

Keyboard Support

The list is scrollable via keyboard. Interactive elements within the list (checkboxes) and the app bar (buttons) are focusable and can be activated with Enter/Space.

Screen Reader Support

The app bar acts as a `

`. The list uses `
    ` and `
  • ` for semantic structure. All interactive elements have descriptive aria-labels. The empty and loading states have appropriate roles or live regions to announce their status.

Focus Management

Focus is managed logically through the app bar and list items. In multi-select mode, the contextual app bar actions are included in the tab order.

Aria Attributes

  • aria-label for icon buttons.
  • role='status' for loading indicators.
  • aria-live for dynamically changing content like the selection counter.

Color Contrast

All text, icons, and status badges meet WCAG 2.1 AA contrast ratios.
