Description

Provides a consistent, centered layout for all public-facing, unauthenticated screens such as Login, Registration, and Forgot Password. It includes a background and a central card to contain the main form or content.

Design System Alignment

This component uses the design system's color palette for backgrounds, spacing tokens for padding and margins, and the card component specification for the central content area, including its shadow and border-radius.

Responsive Specifications

Mobile Adaptations

  • The layout uses vertical and horizontal flex centering to work on any screen size.
  • The content card is full-width with some padding on the smallest screens.
  • Padding inside the card is reduced on viewports smaller than 480px.

Tablet Adaptations

  • The content card maintains its maximum width and remains centered.

Desktop Enhancements

  • The content card maintains its maximum width and remains centered, providing ample negative space.

Accessibility Features

Keyboard Support

The layout itself is not interactive. Keyboard support is delegated to the child components (forms, links) that are placed inside it.

Screen Reader Support

Uses a semantic `

` element to denote the primary content area of the page, which is a best practice for accessibility.

Focus Management

Not applicable at the layout level. Focus is managed by child components.

Aria Attributes

  • No specific ARIA attributes are needed for the layout itself.

Color Contrast

The background and card colors are chosen from the design system's neutral palette to ensure sufficient contrast for any text placed on top.
